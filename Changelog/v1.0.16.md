# Changelog 1.0.16

## ✨ Features

### 新增 `@bugpin/cli` 管理员命令行工具

仓库新增第 4 个 workspace `src/cli/`（TypeScript + Bun），封装 BugPin 所有管理员 REST 端点（约 40 个），覆盖 auth / reports / projects / users / settings / branding 五大组以及本地配置管理。

**亮点**：

- **全局安装**：`cd src/cli && bun link` 后任意目录可调用 `bugpin-cli`
- **仅 cookie session 认证**：与浏览器后台同源，session 持久化到 `~/.bugpin/config.json`（0600 权限）
- **双输出模式**：默认人类可读（list 表格 + 键值对），`--json` 输出原始响应供 jq/脚本解析
- **完整 ID 显示**：list 表格 `id` 列保留完整 36 字符，可直接复制传给 `get/update/delete`
- **multipart 上传**：`reports create` 支持 `--file path1 --file path2` 多次
- **二进制下载**：`reports download` 与 avatar 下载支持任意目录路径
- **本地配置**：新增 `config show / set-url / reset` 子命令，便于新机器/CI 预设服务端 URL
- **错误友好**：401 / 402 / 403 / 网络错误均翻译为可操作提示

**配套 skill**：`src/cli/skills/bugpin-cli/` 指导 agent 高效使用 CLI 并规避常见陷阱（id 截断、字段名差异、jq 写错、EE 端点 402 等）。

**测试**：内置 18 个单元测试覆盖 cookie 提取、错误翻译、tokenize、配置持久化；typecheck 与测试零失败；真实服务端跑通全部查询命令。

## 🔧 Build

- `Dockerfile` builder stage 增补 `COPY src/cli/package.json`，适配新 workspace 拓扑，防止 `bun install` 警告被 `; exit 0` 掩盖而导致后续 build 失败。CLI 本身不进生产镜像。
- 根 `package.json` 新增 `cli` / `typecheck:cli` scripts；`workspaces` 数组加入 `src/cli`。

## 📝 Docs

- `src/cli/README.md`：用法、全局安装、命令清单、错误处理
- `src/cli/skills/bugpin-cli/SKILL.md` + `references/commands.md`：agent 使用指南与完整命令字典
