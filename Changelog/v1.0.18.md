# Changelog 1.0.18

## 🐛 Fixes

### 修复仪表盘"按反馈人统计"把匿名反馈错误归到具名用户名下

`reportsRepo.getStats` 旧实现仅按 `COALESCE(reporter_email, '')` 分组，并用 `MAX(reporter_name)` 选桶名。当数据库中存在大量 `reporter_email` 为空（匿名 / 半匿名）的报告时，它们会被合并到同一个桶，然后随机挑出一个有名字的报告作为整桶代表 —— 直观表现就是仪表盘上某个用户名下出现远超实际的反馈数（例如"贾伟 27 条"），而其中绝大多数其实是其他人或匿名提交的。

修复策略改为按"反馈人身份"优先级分组：

- 有 `reporter_email`：按 `LOWER(email)` 分组（同邮箱视为同一人）
- 无 email 但有 `reporter_name`：按 name 分组（半匿名也是独立反馈人）
- 两者都为空：合并到单一"匿名"桶（前端已有 `dashboard.anonymous` 兜底显示）

新增回归测试 `getStats.byReporter groups anonymous reports separately from named reporters`，构造三类反馈人并断言各自计数独立，避免回归。
